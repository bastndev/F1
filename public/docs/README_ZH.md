**从一个 VS Code 面板启动和管理所有 AI 编程 CLI。**

</div>

---

F1 将 VS Code 变为 AI 编程代理的中心。在嵌入式终端中打开 **Claude Code、Codex、Copilot、Cursor、Kiro、Kilo Code、OpenCode、Grok、Antigravity** — 或任何自定义 CLI —，即时切换，编写更好的提示词，并为每个代理保持项目上下文的低成本加载。

<br>

## 🚀 快速开始

1. 按 **`F1`** 打开 **CLI Hub** 面板。
2. 从启动器中选择一个代理 (或 **自定义 CLI** 以运行您自己的命令)。
3. 从活动栏 (**`Ctrl+3`**) 打开 **My Skills** 以安装或创建 skill。

> 要使用 (**`Alt + E`**) 和 (**`Ctrl + 3`**) ，您需要安装 **[Lynx Keymap Pro](https://open-vsx.org/extension/bastndev/Lynx-Keymap)**。

<h2 align="center"> 𝙵𝚎𝚊𝚝𝚞𝚛𝚎𝚜 </h2>

### 🖥️ CLI Hub

- 在嵌入式 [xterm.js](https://xtermjs.org/) 终端中并行运行多个 AI 编程 CLI。
- 模糊搜索启动器以选择代理；按 **F1** 可直接跳转到面板。
- 每个会话内置的工具：
  -   - **Prompt** — 丰富的编辑器，支持 `@` 文件提及、图片粘贴、skill 芯片、实时拼写标记、提示历史记录 (`↑`)、**PRO / PLAN** 模式，以及发送前的源语言→英文翻译；每个 CLI 会话保持自己的草稿和打开/关闭状态。
  - **翻译** — 内联翻译任何终端选择，自动模式可在其余内容仍在翻译时朗读答案。
  - **使用** — 按 CLI 查看使用情况 / 状态。
  - **快捷键** — 键盘快捷键参考。
  - **命令** — 可搜索的、按 CLI 分类的活动代理斜杠命令面板。
- - **语音** — 朗读回复，加上代理完成时的"叮"声以及 CLI 等待决策（批准、是/否、选择）时的独特确认提示音。
- - **Needs-input badge** — 标签名称旁的一个脉冲圆点（紧凑布局中仅图标徽章）标记任何正在等待您的后台会话。

### 🧩 My Skills

- 从市场安装 skill — **All‑time**、**Trending (24h)**、**🔥 Flame** 和 **Official** 来源。
- 使用引导生成器为 `AGENTS.md`、`CLAUDE.md` 和 `DESIGN.md` 创建 skill，以及按类别的快速模板。
- 按工作区管理和保存本地 skill。

### 🧠 Smart + Skills

- 在启动器中翻转 **Smart + Skills** 开关，然后照常选择代理。
- 启动时，F1 为您的项目构建一个低成本的代码图以及一组内置工作规则，然后让代理读取它们 — 这样它开始时就已经有方向感，并在聊天中回复**"我已准备好工作 ✅"**，无需手动粘贴上下文。
- 由 **My Memory** 驱动：一个快速的纯 TypeScript 引擎，写入结构化的 `.f1/` 项目地图，并保持每个 CLI 的指令文件 (`AGENTS.md` / `CLAUDE.md`) 指向它。临时文件在代理初始化后自动清理。

## 安装

### 方法 1 — 快速打开

根据您的操作系统启动 _Quick Open_：

- <img src="https://www.kernel.org/theme/images/logos/favicon.png" width=16 height=16/> Linux `Ctrl+P`
- <img src="https://developer.apple.com/favicon.ico" width=16 height=16/> macOS `⌘P`
- <img src="https://www.microsoft.com/favicon.ico" width=16 height=16/> Windows `Ctrl+P`

粘贴以下命令并按 `Enter`：

```bash
ext install bastndev.f1
```

### 方法 2 — 扩展视图

1. 打开 Extensions (`Ctrl+Shift+X` / `⌘+Shift+X`)
2. 搜索 **"F1"** (发布者: `bastndev`)
3. 点击 **安装**
