**एक ही VS Code पैनल से हर AI कोडिंग CLI को लॉन्च और प्रबंधित करें।**

</div>

---

F1 VS Code को AI कोडिंग एजेंटों के लिए एक हब में बदल देता है। **Claude Code, Codex, Copilot, Cursor, Kiro, Kilo Code, OpenCode, Grok, Antigravity** — या कोई कस्टम CLI — को एम्बेडेड टर्मिनलों में खोलें, उनके बीच तुरंत स्विच करें, बेहतर प्रॉम्प्ट लिखें, और अपने प्रोजेक्ट के संदर्भ को हर एजेंट के लिए लोड करने में आसान बनाए रखें।

## 🚀 आरंभ करना

1. **CLI Hub** पैनल खोलने के लिए **`F1`** दबाएं।
2. लॉन्चर से एक एजेंट चुनें (या अपना स्वयं का कमांड चलाने के लिए **Custom CLI**)।
3. कौशल स्थापित करने या बनाने के लिए गतिविधि बार से **My Skills** खोलें (**`Ctrl+3`**)।

> (**`Alt + E`**) & (**`Ctrl + 3`**) का उपयोग करने के लिए, आपको **[Lynx Keymap Pro](https://open-vsx.org/extension/bastndev/Lynx-Keymap)** स्थापित करना होगा।

<h2 align="center"> विशेषताएँ </h2>

### 🖥️ CLI Hub

- एम्बेडेड [xterm.js](https://xtermjs.org/) टर्मिनलों में कई AI कोडिंग CLIs को एक साथ चलाएं।
- एजेंट चुनने के लिए फ़ज़ी-सर्च लॉन्चर; पैनल पर सीधे जाने के लिए **F1** दबाएं।
- हर सत्र के साथ अंतर्निर्मित उपकरण:
  - **Prompt** — `@`-फ़ाइल उल्लेख, छवि पेस्ट, कौशल चिप्स, लाइव स्पेल-मार्किंग और भेजने से पहले स्रोत→अंग्रेजी अनुवाद के साथ समृद्ध संपादक।
  - **Translator** — किसी भी टर्मिनल चयन का इनलाइन अनुवाद करें।
  - **Use** — प्रति-CLI उपयोग / स्थिति देखें।
  - **Keymaps** — कीबोर्ड-शॉर्टकट संदर्भ।
  - **Commands** — सक्रिय एजेंट की स्लैश-कमांड पैलेट, खोजने योग्य और CLI-वार।
- **Voice** — उत्तरों को ज़ोर से पढ़ें, साथ ही एक वैकल्पिक "डिंग" जब कोई एजेंट पूरा होता है जबकि आपका ध्यान कहीं और होता है।

### 🧩 My Skills

- मार्केटप्लेस से कौशल स्थापित करें — **All‑time**, **Trending (24h)**, **🔥 Flame**, और **Official** स्रोत।
- `AGENTS.md`, `CLAUDE.md`, और `DESIGN.md` के लिए निर्देशित जनरेटर, प्लस श्रेणी के अनुसार त्वरित टेम्पलेट्स के साथ कौशल बनाएं।
- कार्यक्षेत्र के अनुसार स्थानीय और सहेजे गए कौशल प्रबंधित करें।

### 🧠 Smart + Skills

- लॉन्चर में **Smart + Skills** टॉगल चालू करें, फिर सामान्य रूप से एक एजेंट चुनें।
- लॉन्च पर, F1 आपके प्रोजेक्ट का एक सस्ता कोड-ग्राफ़ और एक सेट अंतर्निहित कार्य नियम बनाता है, फिर एजेंट को उन्हें पढ़ने देता है — ताकि वह पहले से अभिमुख होकर शुरू करे और चैट में **"I am ready for work ✅"** जवाब दे, बिना मैनुअल संदर्भ चिपकाए।
- **My Memory** द्वारा संचालित: एक तेज़, शुद्ध TypeScript इंजन जो एक संरचनात्मक `.f1/` प्रोजेक्ट मानचित्र लिखता है और प्रत्येक CLI की निर्देश फ़ाइल (`AGENTS.md` / `CLAUDE.md`) को उसकी ओर इंगित रखता है। एजेंट तैयार होने पर अस्थायी फ़ाइलें स्वचालित रूप से साफ़ हो जाती हैं।

## स्थापना

### विधि 1 — त्वरित ओपन (Quick Open)

अपने ऑपरेटिंग सिस्टम के आधार पर _Quick Open_ लॉन्च करें:

- <img src="https://www.kernel.org/theme/images/logos/favicon.png" width=16 height=16/> Linux `Ctrl+P`
- <img src="https://developer.apple.com/favicon.ico" width=16 height=16/> macOS `⌘P`
- <img src="https://www.microsoft.com/favicon.ico" width=16 height=16/> Windows `Ctrl+P`

निम्नलिखित कमांड पेस्ट करें और `Enter` दबाएं:

```bash
ext install bastndev.f1
```

### विधि 2 — एक्सटेंशन व्यू

1. एक्सटेंशन खोलें (`Ctrl+Shift+X` / `⌘+Shift+X`)
2. खोजें **"F1"** (प्रकाशक: `bastndev`)
3. **Install** पर क्लिक करें
