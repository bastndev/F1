Overview

One panel for every agent

CLI Hub launches AI coding assistants as real, fully interactive terminals inside your editor — and keeps several running at once. No extra windows, no lost context, no copy-pasting between tools.

On top of the terminals it adds a Prompt composer, a Translator, a Status reader, a Keymaps reference, and My Memory — committed project context every agent reads before it touches your code.

Memory

Give every agent project context

Turn on My Memory and CLI Hub builds a committed .f1/ map of your project. Every CLI you launch reads it first — so agents start with context instead of re-scanning your codebase, saving tokens and time. Because .f1/ is committed, your whole team shares the same context.

FAQ

Common questions

Do I need to install the CLIs separately?

Yes. CLI Hub runs each agent's own command, so the tool has to be on your machine. The launcher shows a green dot when a CLI is ready and a yellow one when it still needs installing.

Can I run more than one agent at the same time?

Absolutely. Each CLI opens in its own embedded terminal. Add a session with Alt + + and switch between them with Tab.

What is the Custom CLI option?

A way to launch any command that isn't in the built-in list — handy for a private agent or a wrapper script. Pick Custom CLI from the launcher and type the command.

How do I see every / command an agent supports?

Press Alt + F1 (or open the Commands tool) for a searchable palette of every slash-command the active CLI ships with — /model, /clear, /help, and so on. Click one to send it.

What does My Memory actually change?

It writes a committed .f1/ project map and points each CLI's instructions file at it, so a freshly launched agent already understands your project instead of scanning every file.

Does the Translator need an API key?

No. Translation uses free providers, and the read-aloud voice runs locally. Nothing to configure and no key required.
