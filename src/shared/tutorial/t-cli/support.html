Welcome

What is CLI Hub?

CLI Hub is a command center for AI coding assistants. It launches and manages multiple terminal CLIs — each in its own embedded terminal — so you can switch agents in a keystroke instead of juggling terminal tabs.

On top of the terminals it adds a Prompt composer, a live Translator, a Status reader, a Keymaps reference, and My Memory — project context every agent can read before it touches your code.

Launch

Open your first CLI

The CLI Hub opens on a fuzzy-search launcher. Pick an agent and it boots straight into an embedded terminal — its real CLI, fully interactive.

  1. Open the CLI Hub panel from the bottom panel area.
  2. Type to search (for example claude or cursor), or press Tab to browse the agent palette.
  3. Press Enter to launch. A green dot means the CLI is installed; a yellow one means it still needs installing.
  4. Need another agent running too? Add a session with Alt + + and hop between them with Tab.
💡

Tip: Need something that isn't in the list? Pick Custom CLI and run any command as your agent.

Tools

Do more without leaving the terminal

The tools menu lives on the right of the panel. Each tool also has a shortcut so you never reach for the mouse.

  • Prompt — a rich composer with @-file mentions, image paste, reusable skills, and automatic Spanish→English before your message reaches the CLI.
  • Translator — select any terminal output and read it in Spanish, with an optional voice that reads it aloud.
  • Status / use — a per-CLI usage and status view for the active agent.
  • Keymaps — the full shortcut reference, always one keystroke away.
💡

Tip: Toggle the Prompt filter with Ctrl + Space to send your message through the composer instead of typing raw into the terminal.

Memory

Give every agent project context

Turn on My Memory and CLI Hub builds a committed .f1/ map of your project. Every CLI you launch reads it first — so agents start with context instead of re-scanning your codebase, saving tokens and time.

  • One toggle — flip My Memory on from the tools menu.
  • The brain button shows freshness at a glance: green is up to date, yellow means your project changed, red means rebuild.
  • Shared by commit — .f1/ lives in your repo, so your whole team's agents share the same context.
💡

Tip: The first build sets up its local engine once per machine. After that, updates are instant.

Shortcuts

Keyboard-first by design

Everything has a key. The essentials, while a session is focused:

  • New session — Alt +
  • Close session — Alt -
  • Next / previous session — Tab / Shift Tab
  • Open CLI selector — CapsLock
  • Toggle Prompt filter — Ctrl Space
  • Prompt · Translate · Status · Keymaps — Shift F1 … F4
💡

Tip: Open Keymaps (Shift F4) any time for the complete, always-current list.

FAQ

Common questions

Do I need to install the CLIs separately?

Yes. CLI Hub runs each agent's own command, so the tool has to be on your machine. The launcher shows a green dot when a CLI is ready and a yellow one when it still needs installing.

Can I run more than one agent at the same time?

Absolutely. Each CLI opens in its own embedded terminal. Add a session with Alt + + and switch between them with Tab.

What is the Custom CLI option?

A way to launch any command that isn't in the built-in list — handy for a private agent or a wrapper script. Pick Custom CLI from the launcher and type the command.

What does My Memory actually change?

It writes a committed .f1/ project map and points each CLI's instructions file at it, so a freshly launched agent already understands your project instead of scanning every file.

Does the Translator need an API key?

No. Translation uses free providers, and the read-aloud voice runs locally. Nothing to configure and no key required.
